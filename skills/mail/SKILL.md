# EC顧客返信スキル

## 概要
EC顧客（楽天市場等）からの問合せに対して、返信メール or SMSメッセージの下書きを生成する。
ユーザーの修正フィードバックを蓄積し、回を重ねるごとにユーザー好みの文体へ近づける。

## 前提ファイル
- **スタイルガイド**: `mail-skill/mail_style_guide.md` — 文体ルール・好みの蓄積
- **例文ペア**: `mail-skill/mail_examples/` — 修正前→修正後のペア保存

（パスのベース: `C:\Users\jl4lv\OneDrive\デスクトップ\0.フジ\900.ClaudeCode\`）

## 実行フロー

### Step 1: パターン選択
AskUserQuestion で以下を選択させる:
- **メール**: 丁寧・フルフォーマット
- **SMS**: 簡潔・短文

### Step 2: 問合せ内容のヒアリング
AskUserQuestion でフリーテキスト入力を受け取る:
- 「お客様からの問合せ内容を貼り付けてください」

追加で必要な情報があれば聞く:
- 返信の要点（伝えたいこと、回答内容）

### Step 3: スタイルガイド＆例文の読み込み
1. `mail-skill/mail_style_guide.md` を Read で読み込む
2. `mail-skill/mail_examples/` 内の最新5件の例文ペアを Read で読み込む
3. これらを下書き生成のコンテキストとして使用する

### Step 4: 下書き生成
以下のルールに従って下書きを生成する:

#### 共通ルール
- 敬語レベル: **非常に丁寧**（最上級の丁寧語・尊敬語・謙譲語を使用）
- 差出人署名: **含めない**（テンプレートで差し込むため）
- スタイルガイドに記載されたルールを**すべて**反映する
- 例文ペアの修正後テキストの文体を参考にする

#### メールの場合
- 件名も生成する
- 挨拶 → 本文 → 締め の構成
- 適度に改行を入れて読みやすくする

#### SMS の場合
- 件名不要
- 簡潔だが丁寧さは保つ
- 長文SMS OK（ただし不必要に長くしない）

### Step 5: ユーザーレビュー
1. 生成した下書きをコードブロックで表示する
2. ユーザーに確認を求める:
   - 「このままでOKですか？修正があれば修正版を貼り付けてください」
   - 選択肢: 「OK（修正なし）」「修正あり」

### Step 6: フィードバック学習（修正ありの場合）
1. ユーザーから修正版を受け取る
2. **差分を分析**して、スタイルルールを抽出する
   - 例: 「〜させていただきます → 〜いたします に統一」
   - 例: 「冒頭の挨拶は省略する」
   - 例: 「商品名は【】で囲む」
3. 抽出したルールを `mail-skill/mail_style_guide.md` に追記する
4. 修正前・修正後のペアを `mail-skill/mail_examples/` に保存する
   - ファイル名: `YYYYMMDD_HHMMSS_{mail|sms}.md`
   - フォーマット:
     ```
     ## 問合せ内容
     （元の問合せ）

     ## 生成版（修正前）
     （Claude が生成した版）

     ## 確定版（修正後）
     （ユーザーが修正した版）

     ## 抽出ルール
     - ルール1
     - ルール2
     ```

### Step 7: 保存（OK の場合）
- 修正なしの場合も例文として保存する（修正前＝修正後として記録）
- 「学習ポイントなし（そのまま採用）」と記録する

## スタイルガイド更新ルール
- 同じルールが複数回出た場合 → 既存ルールの優先度を上げる（★マーク追加）
- 矛盾するルールが出た場合 → 新しい方で上書きし、旧ルールは削除する
- ルールは箇条書きで簡潔に書く

  - `C:/Users/jl4lv/` に一時スクリプトを作成して `python` で実行

# 楽天メルマガ生成スキル

## 概要
楽天市場「制服のフジ」のメールマガジンHTMLを生成するスキル。
自衛隊関連のイベント・記念日をテーマにし、テーマに合った商品を選定して掲載する。

## 前提ファイル
- **CSV**: `003.メルマガGenerator/楽天CSV/` に楽天RMSからダウンロードした item CSV を配置
- **出力先**: `003.メルマガGenerator/メルマガ出力先/`
- **Pythonモジュール**: `003.メルマガGenerator/src/`

（パスのベース: `C:\Users\jl4lv\OneDrive\デスクトップ\0.フジ\900.ClaudeCode\`）

## GUI 共通仕様
- **全GUI は全画面表示**（`self.root.state("zoomed")`）
- **フォント**: メイリオ
- story_selector.py は全ステップ共通の選択GUI（小話・タイトル等）

## 実行フロー

### Step 0: モデル案内
スキル開始時に以下を**テキストで表示**してすぐに Step 1 へ進む（確認待ちは不要）:


### Step 1: CSV 読み込み
以下の Python を実行して商品データを取得する:
```
python -c "import sys; sys.stdout.reconfigure(encoding='utf-8'); sys.path.insert(0, r'C:\Users\jl4lv\OneDrive\デスクトップ\0.フジ\900.ClaudeCode\003.メルマガGenerator\src'); from csv_reader import load_products; products = load_products(); print(f'商品数: {len(products)}'); [print(f'  {p.name[:40]} / {p.id}') for p in products[:3]]"
```
楽天CSVフォルダ内の最新CSVを自動検出して読み込む。

### Step 2: テーマ検索・提案
1. **WebSearch** で **当月＋翌月** の両方を対象に検索する:
   - 「自衛隊 イベント {今月} {今年}」
   - 「自衛隊 イベント {来月} {今年}」
   - 「海上自衛隊 艦艇公開 {今月} OR {来月} {今年}」
   - 「自衛隊 記念日 {今月} {来月}」
   - 「海上自衛隊 呉 広島 イベント {今月} OR {来月} {今年}」
   ※ {来月} が1月になる場合は {今年+1} を使う
2. 検索結果をもとに **20件のテーマ候補** を提示（当月・翌月を混在可）
3. 各テーマには以下を含める:
   - テーマ番号（1〜20）
   - テーマ名（例:「護衛艦かが就役記念」）
   - 簡単な説明（1〜2行。イベント内容や背景を簡潔に）
   - 日付・カテゴリ（記念日 / 就役日 / イベント / 新商品 / 季節）
4. **並び順ルール**:
   - **日付が近い順** に番号を振る（最も近いイベントが1番）
   - **広島県・呉市に関連するイベント** や **呉基地所属艦艇** に関するイベントは優先的に上位に表示する
   - 過去の日付のイベント（記念日・歴史テーマ）は下位に配置する
5. **優先度ルール**:
   - **高**: 護衛艦・潜水艦等の就役記念日、進水記念日、艦艇の節目（○周年）、艦艇一般公開
   - **中**: 記念日（海上自衛隊の日等）、航空祭・航空ショー、歴史テーマ
   - **低**: 基地祭・駐屯地イベント（さくらまつり、見学ツアー等）
6. **商品関連フィルタ（必須）**:
   - テーマ候補を提示する前に、CSVの商品名を検索し、そのテーマに関連する商品が存在するか確認する
   - 関連商品が **1件も見つからないテーマは候補に含めない**
   - 例: 「護衛艦いかづち」→ 商品名に「いかづち」「むらさめ」を検索
   - 例: 「ブルーインパルス」→ 商品名に「ブルーインパルス」「T-4」を検索
   - 汎用的な自衛隊グッズ（海自グッズ全般等）でカバーできるテーマはOKとする

### Step 3: テーマ選定（ユーザーインタラクション）
ユーザーに以下の選択肢を提示する:
- **1〜20**: 提案テーマから番号で選ぶ
- **テキスト入力**: 自分のアイデアを入力 → AIがアイデアを取り込み再提案
- **再提案**: 別の角度からテーマを再検索

**アイデア注入ループ**:
ユーザーがテキストでアイデアを入力した場合:
1. ユーザーの入力内容を考慮してテーマを再検討
2. WebSearch で追加検索（必要に応じて）
3. 新しいテーマ候補を20件再提示
4. ユーザーが満足するまでこのループを繰り返す

### Step 3.5: 小話選定（GUI）
「▼▼ おすすめ商品はコチラ ▼▼」の上に、テーマに関連する小話を挿入する。

**Phase 1: 小話テーマ選択**
1. 選択されたメルマガテーマに基づき、Claude が5つの小話テーマ候補を作成する
   - 例: 部隊の歴史、艦艇の活躍、今後の出港予定、装備の解説、逸話・エピソード
2. 候補を JSON ファイル（文字列リスト形式）に書き出す
3. story_selector.py を起動して GUI でテーマを1つ選ばせる:
```
python story_selector.py --options-file themes.json --title "小話テーマ選択" --header "★ 小話のテーマを選んでください ★" --no-preview
```
4. stdout JSON の selected_index を取得

**Phase 2: 小話本文選択**
1. 選ばれたテーマに基づき、Claude が5つの小話本文を作成する（各3〜5行）
2. 候補を JSON ファイル（文字列リスト形式）に書き出す
3. story_selector.py を再度起動して GUI で本文を1つ選ばせる:
```
python story_selector.py --options-file stories.json --title "小話選択" --header "★ メルマガに掲載する小話を選んでください ★"
```
4. stdout JSON の selected_text を取得し、generate_newsletter の story 引数に渡す

**小話のガイドライン:**
- 3〜5行程度の短い読み物
- 絵文字は使わない（★ ♪ ▼ 等の記号装飾のみ）
- テーマに関連する部隊・艦艇の歴史、活躍、今後の予定などを題材にする
- 親しみやすいトーンで、お客様が読んで楽しめる内容にする
- WebSearch で事実確認すること

### Step 4: 商品選択（GUI - 画像グリッド）
1. 選択されたテーマに関連するキーワードを抽出
2. csv_reader で全商品をロードし、search_products でキーワード絞り込み
3. Claude がテーマとの関連度を判断し、**48件に厳選** して JSON ファイルに書き出す（**関連度の高い順にソート**）
   - 艦艇名・部隊名に直接関連する商品を優先（左上に配置される）
   - **必須固定枠**: 商品ID （訳ありカレー2食セット 1,000円）を **必ず1件** 含める
   - **食品類（カレー・菓子・食品等）を最低5件** 含める（上記固定枠を含む）
   - **タオル・手ぬぐい・ハンカチ類を最低5件** 含める
   - 汎用的な海自グッズ・自衛隊グッズで補完
   - 在庫2以下の商品は除外する
4. product_selector.py を起動して GUI でユーザーが **12件をクリック選択** する（候補は48件表示）:
```
python product_selector.py --products-file /path/to/filtered.json
```
5. stdout の JSON から selected_product_ids を取得（**選択順 = メルマガ掲載順**）
6. cancelled=true の場合はユーザーに確認して再実行 or 中止

**商品選択 GUI 仕様:**
- **動的グリッドレイアウト**: 画面幅に応じてカラム数を自動計算（余白なし）
  - `CELL_WIDTH=150px` を基準に `(画面幅 - 40) // (CELL_WIDTH + CELL_PAD)` で列数決定
- 各セル: 商品画像サムネイル（110x110px） + 商品名（小さい文字） + 価格【在庫min-max】
  - 在庫数は倉庫=1のSKUを除外したmin-max表示（例: 【3-15】、単一SKUは【5】）
- **クリックで選択**: クリックした順に赤い番号バッジをオーバーレイ表示
- 選択した順序 = メルマガの左上から右下への掲載順序
- 再クリックで選択解除（番号自動振り直し）
- 画像はバックグラウンドスレッドで非同期ダウンロード（`_product_cache/` にキャッシュ）
- Pillow 必須（画像表示用）
- 「全解除」「確定」「キャンセル」ボタン
- **補正テキストボックス**（GUI下部）:
  - テキスト入力欄 + 「再選定」ボタンを配置
  - ユーザーが補正コメント（例:「食品をもっと入れて」）を入力して再選定ボタンを押すと、GUI を閉じて `{"reselect": true, "comment": "..."}` を返す
  - コメント空欄で再選定 → ニュートラル再選定（前回と異なる商品構成にする）
  - Claude は comment を参考に商品を再選定し、product_selector.py を再起動する

### Step 5: カテゴリバナー（自動・全選択）
GUI は起動しない。全15件のバナーを自動的に選択する:
```python
banner_indices = list(range(15))  # 0〜14 全選択
```
- banner_selector.py は起動不要
- 将来的に選択を変えたい場合はこのステップを復元する

### Step 5.5: メルマガタイトル生成（GUI）
Claude がテーマに基づき、メルマガのメール件名（タイトル）を生成する。
1. テーマ名・小話・選択商品を考慮して、開封したくなるタイトルを **10候補** 作成する
2. 候補を JSON ファイル（**文字列リスト形式**: `["タイトルA", "タイトルB", ...]`）に書き出す
3. story_selector.py を起動して GUI で1つ選ばせる:
```
python story_selector.py --options-file _mail_titles.json --title "メルマガタイトル選択" --header "★ メール件名を選んでください ★" --no-preview
```
4. stdout JSON の selected_text を取得
5. 選ばれたタイトルは gui.py の第3引数として渡す

**タイトルのガイドライン:**
- 30文字以内（楽天RMSの件名上限を考慮）
- 記号装飾を使う（★ ♪ ▼ 等）
- テーマの核心を端的に伝える
- お得感・限定感・時事性を意識する
- 絵文字は使わない

### Step 6: HTML 生成
csv_reader.py で商品データを取得し、html_generator.py でHTMLを生成する。
テーマタイトル・導入文・商品リスト・カテゴリバナー・小話を引数として渡す。

生成コマンド:
```
python html_generator.py --products "JSON" --title "テーマ名" --intro "導入文" --banners true --story "小話テキスト" --banner-indices "[0,1,2,...]"
```

### Step 7: GUI 表示・出力
1. 生成したHTMLを `_latest_merumaga.html` にも書き込む（次回デフォルト読み込み用）
2. GUI 起動時は **必ずファイルパス・テーマ名・メルマガタイトルを引数で渡す**:
```
python gui.py "出力先/YYYYMMDD_テーマ名.html" "テーマ名" "メルマガタイトル"
```
3. 引数なしで起動すると `_latest_merumaga.html`（古い内容の可能性あり）を読むため注意

**プレビューGUI 仕様:**
- ウィンドウサイズ: 1300x750px
- **メール件名バー**（黄色背景）: タイトルをクリックでクリップボードにコピー
- HTMLプレビューエリア（横スクロールバー付き、編集可能）
- 「クリップボードにコピー」ボタン（HTML本文をコピー、ダイアログなし・ステータスバー表示のみ）
- 「ファイルに保存」ボタン
- 保存ファイル名: `YYYYMMDD_テーマ名_メルマガタイトル.html`

## テーマ導入文の生成ガイドライン
- サンプルメルマガのトーンを参考に（親しみやすく、記号装飾を使う）
- テーマの記念日・イベントに触れつつ、商品への誘導を自然に行う
- 文量は3〜5行程度
- 絵文字は使わない（★ ♪ ▼ 等の記号装飾のみ）

## HTML デザインルール
- **導入文**: font size="4" で太字なし（b タグ不使用）。size=5以上だと不自然な改行が発生する
- **タイトル**: font size="6" + b タグで太字あり
- **「おすすめ商品はコチラ」**: font size="6" + b タグで太字あり
- テーマセクションのテーブル幅: 600px（ヘッダーテーブルは700px）
- 商品グリッド: 3列テーブル（12件 = 3列×4行）

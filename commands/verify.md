コーディング後の検証を一括で実行する skill。

対象ファイルのパスを指定して使う（例: `/verify src/app.py`）。
パス未指定の場合は、直前に作成・変更したファイルを対象とする。

## 実行手順（すべて自動で順に実行すること）

### Step 1: CLAUDE.md 必須検証
1. 対象ファイルの先頭 20 行を読み、import 一覧を確認する
2. 実際に import して動作確認する（ast.parse ではなく実 import）:
   ```
   python -c "import sys; sys.path.insert(0, r'<対象ディレクトリ>'); from <モジュール> import <関数>; print('OK')"
   ```
3. パス定数が実在するか確認する:
   ```
   python -c "from pathlib import Path; p=Path(r'<パス>'); print(p, p.exists())"
   ```

### Step 2: コードレビュー
以下の観点でチェックし、問題を報告する:
- バグ・論理ミス
- セキュリティ上の問題
- パフォーマンスの問題
- 可読性・命名
- 保守性・モジュール構成
問題は重大度の高い順に報告する。

### Step 3: pytest テスト生成・実行
1. 対象コードに対して pytest テストを生成する
2. テストファイルを書き出す
3. `python -m pytest <テストファイル> -v` で実行する
4. 失敗があれば原因を分析する

## 出力フォーマット

結果を以下の形式で報告すること:

```
## 検証結果

### import 確認: ✅ OK / ❌ NG
（詳細）

### パス確認: ✅ OK / ❌ NG / ⏭ 該当なし
（詳細）

### コードレビュー: ✅ 問題なし / ⚠ N件の指摘
（指摘一覧）

### テスト結果: ✅ N passed / ❌ N failed
（詳細）
```

## 注意事項
- 問題が見つかった場合、修正案を提示するが自動修正はしない
- ユーザーの承認を得てから修正する
- 回答はすべて日本語で行うこと
